# ブラウザ完結でデータを送らない日本語業務ツール集「CODE:LIFE Tools」を作っています

> [!NOTE]
> 本ファイルは Zenn / Qiita / codelife.cafe への転載用記事ドラフトです（サイトのビルド対象外）。
> 公開・投稿は人間が最終確認のうえ実施してください。

**要点: [CODE:LIFE Tools](https://tools.codelife.cafe) は、CSV文字化け修復や全角半角変換など42種の業務向けWebツールを、すべてブラウザ内の処理だけで提供する無料・オープンソースのツール集です。** 入力したデータをサーバーに送信しない設計のため、業務データを外部に出さずにその場で処理できます。

## こんな「困った」ありませんか

- 取引先から届いたCSVをExcelで開いたら文字化けしていた
- 顧客名簿のカタカナや英数字が全角・半角バラバラで集計できない
- APIから取得したJSONを、Excelで扱えるCSVに変換したい
- 郵便番号だけのリストに住所を入れる作業を延々と手でやっている

どれも数分で終わるはずの作業ですが、社外のWebツールに業務データを貼り付けるのは気が引ける——そんな場面は多いはずです。

## Q. なぜ「データを送らない」ことが業務で重要なのか

サーバー側で処理するWebツールでは、入力データが外部に送信される場合があります。便利な一方で、顧客名簿・取引データ・個人情報を含むファイルを扱う場合、社内のセキュリティポリシーや個人情報保護の観点から利用をためらう（あるいは禁止されている）ケースが少なくありません。

CODE:LIFE Tools は全ツールが**完全クライアントサイド処理**です。変換・計算・画像加工などの処理はすべてお使いのブラウザ内（JavaScript / WebAssembly / Web Worker）で完結し、入力データが外部サーバーへ送信されることは一切ありません。Cookieなし・個人追跡なし・広告なしで運営しており、PWAとしてインストールすればオフライン環境でも利用できます（ブラウザ内AIを使う一部ツールは、初回利用時のみモデルファイルの取得が必要です）。

## Q. 実際にどんなツールがあるのか — 代表5本

### 1. CSVの文字化けを直したい → [CSV文字化け修復](https://tools.codelife.cafe/csv-fixer)

Shift_JISのCSVをExcelやmacOSで開いて文字化けした、というときに。文字コード（Shift_JIS / UTF-8 / EUC-JP等）を自動判定し、Excelでそのまま開けるBOM付きUTF-8に変換します。ファイルはブラウザ内で読み込まれるだけで、どこにもアップロードされません。

### 2. 全角・半角の表記ゆれを統一したい → [全角⇔半角変換](https://tools.codelife.cafe/zenkaku-hankaku)

カタカナ・英数字・記号の全角半角を一括変換。カテゴリごとに「カタカナは全角、英数字は半角」のような細かい制御ができるので、名簿やマスタデータの表記統一に向いています。

### 3. JSONとCSVを行き来したい → [JSON ↔ CSV 変換](https://tools.codelife.cafe/json-csv)

APIレスポンスのJSONをCSVに、あるいはCSVをJSONに相互変換。ネストしたオブジェクトの展開や型推論、Excel文字化け対策のBOM付きUTF-8出力に対応しています。

### 4. 郵便番号から住所を一括で埋めたい → [郵便番号→住所変換](https://tools.codelife.cafe/zipcode)

7桁の郵便番号から住所を検索・一括変換。Excelからの貼り付けやCSV出力に対応しています。住所データは日本郵便の郵便番号データを静的ファイルとしてサイトに同梱しているため、検索時も入力データは外部に送信されません。

### 5. 税込・税抜をさっと計算したい → [消費税・税込計算](https://tools.codelife.cafe/tax)

税込⇔税抜の即時計算。軽減税率（8%）や過去税率（3% / 5% / 8%）、端数処理（切り捨て / 四捨五入 / 切り上げ）に対応しているので、過去の伝票確認や見積もり時の検算にも使えます。

このほか、テキスト差分・正規表現テスター・PDF結合/分割・画像圧縮・背景削除（ブラウザ内AI推論）など、全42種のツールを公開しています。一覧は [https://tools.codelife.cafe](https://tools.codelife.cafe) からどうぞ。

## Q. 本当にデータを送っていないのか、どう確認できるのか

CODE:LIFE Tools は**オープンソース**です。すべてのコードを [GitHub（maru0014/tools-codelife-cafe）](https://github.com/maru0014/tools-codelife-cafe) で公開しており、処理内容は誰でも確認できます。ブラウザの開発者ツールでネットワークタブを開いたまま操作すれば、入力データが送られていない挙動を確認できます。コード公開とあわせて、検証の材料にしてください。

なお、サイトの改善のためにCookieを使わない匿名・集計ベースの利用統計は取得していますが、入力データの中身や個人を特定する情報は含まれません。詳細は[プライバシーポリシー](https://tools.codelife.cafe/privacy)にまとめています。
